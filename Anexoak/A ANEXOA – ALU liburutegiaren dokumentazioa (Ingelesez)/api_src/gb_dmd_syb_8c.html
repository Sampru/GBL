Detailed Description

Database access specific functions for SYBASE DBMS.

Acronyme
GBdmdSyb

Database access specific functions for SYBASE DBMS.

  • File that contains specific functions for access with SQL sentences to a SYBASE DataBase Management System. Important: they will be implemented directelly with the functions existing in gb.c/gbodbc.c, i.e. they will not be directly call to the lower libraries (client-library of sybase, odbc, ...)

  • Also, this module contains DB related functions with SYBASE specifics. No portability to other DBMSs is expected. (SYBxxx)

    • Sybase locking mechanisms behaviour: We've got two table locking types:

      SHARED LOCK: syb_lck_shared Other users can read, but can't write. Other users can adquire shared locks, but not exclusive locks. We'll use this lock table when we are not going to change anything in the table we are locking, but we want to prevent other transactions to modify until we finish ours.

      EXCLUSIVE LOCK: syb_lck_exclusive Other users can't read nor write until we finish our transaction. Other users cannot adquire any type of lock. We'll use this lock table when we are going to change something on the table.

    All the locks in version 11.9.2 and previous are still equivalent to a: LOCK TABLE ... IN EXCLUSIVE MODE

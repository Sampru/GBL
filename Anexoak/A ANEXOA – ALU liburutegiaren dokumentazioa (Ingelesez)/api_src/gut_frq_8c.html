Check if a task must be launched in a date or not according to the launching frequency.

Check if a task must be launched in a date or not according to the launching frequency

  • There are some frequencies that make reference to end of month, end of quarter,... To know if they mean open dates or every date, check if the batch is launched every day or only on open dates.

  • If there is a specific list of dates (LdDatFrq) these dates must be added to the list of valid (or invalid) dates because of the frequency itself.

    Parameters
    CtxDatabase context
    DatBatch date
    BatBatch identifier
    NamTask identifier
    FrqTask launching frequency. See enum GutLchFrq for details
    LdDatFrqList of dates for using in those frequencies which deal with a list of dates. In case it is NULL, the list of dates will be that configured for the user for the batch and task passed as parameter.
    FrqFlg(OUT) Checking flag result: GutFlgErr = -1 (Not valid), GutFlgOk = 0 (valid).
    Returns
    Error code (0: OK)

Checks if a date is a week day (moday - friday, for Muslim week Sunday - Thursday)

Checks if a date is a week day (moday - friday) Saturday (6) and Sunday (7) are weekends days. But for some Muslim countries Firday (5) and Saturday (6) are the weekends days (calendar is_msl_wek_cty function says a country is a Muslim weekend country)

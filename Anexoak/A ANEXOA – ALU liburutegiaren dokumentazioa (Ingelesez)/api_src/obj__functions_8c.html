Detailed Description

Object manipulation.

Header file
obj_def.h
Acronyme
OBJ (OBJect manipulation)

Generic object container (object list) manipulation.

  • An object list is a list, LdXxx, containing objects (frequently lists themselves but also pointers to strings) accessible by name (object code). Each object will optionally contain its destructor.

    • Destructors. Each object contains a destructor, a function that must be specified while setting it. If a NULL is given as destructor it won't be called. This destructor will be called with only one parameter, a pointer to the object and returns an error code. Its prototype is: 
      int WHAteverDestructor (void **Ld);
    • The return the error code must be 0 when the operation has been correctly carried out. Otherwise, any operation calling it is aborted.

    • Examples of destructors are:

      • DYNdetruitListDyn
      • OBJfreePtr (equivalent to the macro MMfreePtr, see details)
      • MYdetruitLdLdEch (specific)
      • NULL

    • A new destructor to call, for example, to the functions DYNdetruit2ListDyn could be:

      int MYdetruitLdLdEch (void **Ld) { return DYNdetruit2ListDyn (Ld, "LdEch"); }

    • Initialisation. To initialise of the object list LdXxx (necessary) we do:

      void *LdXxx = NULL;

    • Setting an element. To set an element LdEle, with code "Ele" we must provide its destructor. As for the example this element is a list, we provide a standard destructor of the object, DYNdetruitListDyn:

      OBJsetVal (&LdXxx, LdEle, DYNdetruitListDyn, "Ele", "ElTyp", "Element comment");

    Of course, the prototype of the destructor must be accessible because otherwise we would have an undefined variable.

    Should an object with the same code already exist, this will be destroyed (or deleted) beforehand with its own destructor (which was specified while setting it).

    • Getting an element. We get a pointer to the object (which shouldn't be destroyed externally):

      OBJgetVal (LdXxx, "Ele", &LdEle);

    • Delete an element. Removes the element from the object list calling its destructor:

      OBJdelObj (LdXxx, "Ele");

    • Destroy the object list.

      OBJdestroy (&LdXxx);

    • Example:

      • void *LdObj;
      • ...
      • ERRchk (OBJsetVal (&LdObj, LdVal, DYNdetruitListDyn, "dev", "Ld", "Currency list"));
      • ...
      • ERRchk (OBJsetVal (&LdObj, STRdup ("ESP"), OBJfreePtr, "dev", "char", "Currency string"));
      • ERRchk (OBJdelObj (LdObj, "dev"));
      • ERRchk (OBJsetVal (&LdObj, "EUR", NULL, "dev", "char", "Currency string"));
      • ERRchk (OBJdestroy (&LdObj));

Initialise LdObj.

Initialise LdObj. The initialised object will always have the columns:

  • - "cod": object code
  • - "del": pointer to the function to destroy the object
  • - "typ": object type
  • - "com": object comments
  • and apart from these ones, in the object list it will be added a column for each different object type required.
  • This function will first check in the object list is initialised and then if a column for the required object type exists.
Parameters
LdObj(in/out) Object general list
TypType of the required object. This parameter supports one of these values:
  • OBJ_BIT: bit
  • OBJ_TNY: tiny
  • OBJ_SML: small
  • OBJ_INT: integer
  • OBJ_FLT: float
  • OBJ_DAT: date
  • OBJ_STR: string
  • OBJ_VOD: dynamic list (void *)
Returns
Error code (0: OK)

Extract some columns from some objects of a list of objects.

Extract some columns from some objects of a list of objects. We may specify which columns are extracted or specify 0 columns, which means all. We may specify which objects are extracted or specify 0 object codes, which means all. A specified object needs not exist. This means that the resulting list may be NULL (if no object is found). When we specify "all columns" the resulting list will have the union of all columns of all the objects specified (which don't need to exist all on all objects). When we specify a list of columns explicitly, all columns must exist on all the specified objects.

LdOut(out) Dynamic list with the specified objects
NbColNumber of columns to extract (0: All)

  • ClnN: Column name NbCol
  • NbCod Number of object names (0: All)
  • Cod1: Object code 1
  • Cod2: Object code 2
  • CodM: Object code NbCod

Set an object.

Set the pointer to an object. Objects are unique. If the object list is null its internal structure will be initialised.

  • If the object exists and the new one is different to the existent one, before to insert it, it removes the existent one. Otherwise, it does nothing.
  • IMPORTANT!!: It is very important to give the correct object type specially when adding object which value is a number, because otherwise, the object will not be managed correctly.
Parameters
LdObj(in/out) Object list
ObjObject value:
  • In case this object is a number (bit, tiny, small, integer, float) this value must be directly the number value, not a pointer
  • In case this object is a date or a string, this value must be directly the string value (char *)
  • In case this object is a dynamic list, this value must be the list pointer (void *)
DelObject destructor (function). If it only useful for pointer type objects
CodObject code
TypObject Type. This parameter can be one of the defined object types or any string the user wants. In this last case, the object type will be consider as OBJ_VOD. The predefined object types are:
  • OBJ_BIT: bit
  • OBJ_TNY: tiny
  • OBJ_SML: small
  • OBJ_INT: integer
  • OBJ_FLT: float (Not managed by the moment)
  • OBJ_DAT: date (pointer)
  • OBJ_STR: string (pointer)
  • OBJ_VOD: dynamic list (void *) (pointer)
ComObject comment
Returns
Error code (0: OK)

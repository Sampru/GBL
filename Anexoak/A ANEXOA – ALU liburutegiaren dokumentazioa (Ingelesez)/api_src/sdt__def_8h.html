Carry out calculations.

Carry out calculations for a field if a calcul group is defined. It verifies that a calcul can be launched and it launch them. The calculations are launched in the next cases:

  • Mode entry: always
  • Mode modify: only when a static data is read, that is, in selected or modified phases The calculations are launched in loops, that is, each loop it launches the specific calculations that are possible at that time.
Parameters
CtxDatabase access context
LdObjObjects list
LdSdtColumn data descriptor (e.g. label, db column name, ...)
ColColumn index
LinLine index
Returns

Algorithm
  • If there aren't calculations defined exit
  • Check the window's status, if it is not in entry mode or modify mode (selected/modified phases), it goes out
  • Initialise the list to save the calculated fields
  • Before launching the calcules, invalidate the fields that are calculed with the modified field (and others)
  • Launch the calculation function of the possibles functions in each moment.

Format a datum with some characteristics.

  • Doubles:
    • Format: Use of commas and points or points and commas. The default is in cst.ide = 'DblFormat' FMT_AME (0): American, formatted, e.g. 3,159,124.4238 FMT_EUR (1): European, formatted, e.g. 3.159.124,4238 FMT_AME_NO_THO_SEP (2): American, no separator for thousands, etc e.g. 3159124.4238 FMT_EUR_NO_THO_SEP (3): European, no separator for thousands, etc e.g. 3159124,4238
    • NbDec: Number of decimals. The default is in cst.ide = 'DblNbDec'
    • Method: Rounding method. The default is in cst.ide = 'DblRound' 0: Round to the nearest, 9.231 -> 9.23, 9.236 -> 9.24, 9.235 -> 9.24 1: Truncate, 9.231 -> 9.23, 9.236 -> 9.23, 9.235 -> 9.23 2: Up next, 9.231 -> 9.24, 9.236 -> 9.24, 9.235 -> 9.24
    • Remove Zeroes: Indicates that the zeroes are been removed. The default is in cst.ide = 'DblRmZer'
  • Dates:
    • Method: Method used. This Method is used to complete the date 1 = date <= refDate 2 = date >= refDate 3 = date > refDate
    • Format: Date style. Is not parametreable. It uses the DTgetFmt () function to get the format. 5 = European 10 = American

Validation of the introduced data.

Validate and complete the introduced data

  • Doubles:
    • Format: Use of commas and points or points and commas. The default is in cst.ide = 'DblFormat'
      • 0: American, formatted, e.g. 3,159,124.4238
      • 1: European, formatted, e.g. 3.159.124,4238
      • 2: American, no separator for thousands, etc e.g. 3159124.4238
      • 3: European, no separator for thousands, etc e.g. 3159124,4238
    • NbDec: Number of decimals. The default is in cst.ide = 'DblNbDec'
    • Method: Rounding method. The default is in cst.ide = 'DblRound'
      • 0: Round to the nearest, 9.231 -> 9.23, 9.236 -> 9.24, 9.235 -> 9.24
      • 1: Truncate, 9.231 -> 9.23, 9.236 -> 9.23, 9.235 -> 9.23
      • 2: Up next, 9.231 -> 9.24, 9.236 -> 9.24, 9.235 -> 9.24
    • Remove Zeroes: Indicates that the zeroes are been removed. The default is in cst.ide = 'DblRmZer'
  • Dates:
    • Method: Method used. This Method is used to complete the date
      • 1 = date <= refDate
      • 2 = date >= refDate
      • 3 = date > refDate
    • Format: Date style. Is not parametreable. It uses the DTgetFmt () function to get the format
      • 5 = European
      • 10 = American
Parameters
LdObjObjects list.
LdSdtDescriptor data
LineIndex of the line of LdSdt
InxValIndex of the line of data of LdSdt
Returns
Error code (0 if OK).

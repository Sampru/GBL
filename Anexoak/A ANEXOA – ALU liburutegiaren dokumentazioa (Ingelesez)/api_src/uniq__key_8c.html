char *  Nam,

Reserve Nbk unique key numbers for a given table.

Reserve Nbk unique key numbers for a given table.

  • This table needs not be initialised for each new key type.
  • If the table is not initialised, the first key given will be 1.
  • It is recommended that the name of each key is the name of its table or the name of the table + name of the field.
  • This operation is carried out within a transaction. If the calling funtion is already within a transaction this must be done through the nested transaction schema DBsetNestedTrans.
  • In order to avoid blocking other treatments it is better to reserve this number before starting the transaction of a treatment to leave free access to this table.

  • The reserved idx numbers will be Idx, Idx + 1, Idx + 2, ..., Idx + Nbx - 1. For example, if we reserve 5 and the function sets Idx = 8 we'll have reserved (8, 9, 10, 11, 12).

Reserve Nbk unique key numbers for a given table.

Reserve Nbk unique key numbers for a given table. The key can be in a different book, so that is way the 'Als' parameter is provided.

  • This table needs not be initialised for each new key type.
  • If the table is not initialised, the first key given will be 1.
  • It is recommended that the name of each key is the name of its table or the name of the table + name of the field.
  • This operation is carried out within a transaction. If the calling funtion is already within a transaction this must be done through the nested transaction schema DBsetNestedTrans.
  • In order to avoid blocking other treatments it is better to reserve this number before starting the transaction of a treatment to leave free access to this table.

  • The reserved idx numbers will be Idx, Idx + 1, Idx + 2, ..., Idx + Nbx - 1. For example, if we reserve 5 and the function sets Idx = 8 we'll have reserved (8, 9, 10, 11, 12).

ALUMCR_EXP int EMLsrvCnf ( void *  LdEmlSrv)

Confirm the e-mail server/user/password data.

Confirm the e-mail server/user/password data. A encrypted file will be generated with the information entered by the user and that is passed as parameter in a list. This list must contain at list the following columns:

  • - Ide: identifier for the e-mail server configuration
  • - Srv: server name
  • - Ath: authentication in server (Y, N)
  • - Usr: user name
  • - Pwd: password
  • The encrypted file will be generated in "FXT_EXPORT"/EmlSrv directory
Parameters
LdEmlSrvE-mail server definition list
Returns
Error code (0: OK)

Send an e-mail message.

Send an e-mail message. The e-mail message will be sent by the user indicated in the parameters, with the e-mail server and address defined for him.

  • The command used to send the e-mail message must be configured in 'Commands' table for the treatment:
  • - EmlAth: if the e-mail server that is being used needs authorization, this is, user/password
  • - EmlNoA: if the e-mail server that is being used does not needs authorization
  • If several servers are used and some needs authorization and others not, both treatments must be configured in 'Commands' table.
  • The command configured must have obligatory the following parameters and in this order:
  • - Source address
  • - Destination address
  • - E-mail server
  • - User for authorization in e-mial server. Only if authorization is needed
  • - Password for authorization in e-mial server. Only if authorization is needed
  • - Subject for the message
  • - Body for the message
  • - Attached file. This parameter must be optional in the command
Parameters
CtxDatabase connection
UsrApplication user who sends the e-mail message
DstAdrDestination address who receives the message. It can be several address separated by ',' or ';'
TitMessage subject. It can NOT be empty or NULL. It is obligatory
BdyMessage body. It can NOT be empty or NULL. It is obligatory
FilFile to attach in the message. It can be empty or NULL. it is optional
SndUsrFlag indicating if the sender address must be included (1) in the destination address or not (0)
Returns
Error code (0: OK)

Send an e-mail message for a group of addresses.

Send an e-mail message for a group of addresses. The e-mail message will be sent by the user indicated in the parameters, with the e-mail server and address defined for him.

  • In the given destination group there can be users, so that in that case the message will be sent to the e-mail address configured for that user, or there can be e-mail addresses explicitely.
  • An unique e-mail message is sent to all the destinations in the group
  • The command used to send the e-mail message must be configured in 'Commands' table for the treatment:
  • - EmlAth: if the e-mail server that is being used needs authorization, this is, user/password
  • - EmlNoA: if the e-mail server that is being used does not needs authorization
  • If several servers are used and some needs authorization and others not, both treatments must be configured in 'Commands' table.
  • The command configured must have obligatory the following parameters and in this order:
  • - Source address
  • - Destination address
  • - E-mail server
  • - User for authorization in e-mial server. Only if authorization is needed
  • - Password for authorization in e-mial server. Only if authorization is needed
  • - Subject for the message
  • - Body for the message
  • - Attached file. This parameter must be optional in the command
Parameters
CtxDatabase connection
UsrApplication user who sends the e-mail message
GrpGroup of addresses for which the message must be sent. by ',' or ';'
TitMessage subject. It can NOT be empty or NULL. It is obligatory
BdyMessage body. It can NOT be empty or NULL. It is obligatory
FilFile to attach in the message. It can be empty or NULL. it is optional
SndUsrFlag indicating if the sender address must be included (1) in the destination address or not (0)
Returns
Error code (0: OK)

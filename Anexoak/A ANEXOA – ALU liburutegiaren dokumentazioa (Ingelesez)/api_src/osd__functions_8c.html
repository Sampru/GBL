int ALERTlstGen ( void *  LdMsg)

Launch an alert message with the containing of a list.

Launch an alert message with the containing of a list. The list must have a column 'msg' where the messages are stored. If the list has more the 15 messages an alert message will be for each group of 15 messages, this is, for example if the list contains 32 messages, 3 alert messages will be launched.

IMPORTANT: this function should not be called from applications that run in background, as batch or WBS agents. Because the alert is shown in a server and no one can confirm the alert.

Parameters
LdMsgList with the messages
Returns
Error code. 0: Ok
Author
Josune BIKUNA - 19/Sep/2002
int get_files_type ( char *  Path,
void *  Ld 
)

Add file access information to a filename list.

Given a filename list (column name: "file") and the path where these files have been obtained get the file access information. This file list will normally have been obtained through the function OSDgetPathFiles (). The information added is the mode and the file type. The access column will be called "mod". The type column will be called "typ". Types may be checked through the standard macros (<sys/stat.h>):

  • S_IFREG: regular
  • S_IFDIR: directory The type column is suitable for DYNwhere () execution.
Parameters
PathDirectory where the files are to be looked for
Ld(in/out) Dynamic list containing a list of files, normally generated by OSDgetPathFiles ().
Returns
Error code (0: OK)

int OSDgetPathFiles ( char *  Path,
char *  Pattern,
void **  LdFil 
)

Get list of a files of a list of paths that match a pattern.

Get list of a files of a list of paths that match a pattern

  • The directories of the path must be separated by a ";"
  • The list of paths must have at least one column "dirname" with them.
  • This function returns a list with two columns ("file", "dirname") that will contain the recupered files.

  • In PCVER, if the pattern is of the form *.txt (with extension of 3 characters long) it will return all files whose extension starts with txt, even if the extension is longer. The pattern *.txt has same result than *.txt*

    Parameters
    PathDirectory where the files are to be looked for
    PatternFiltering pattern (regular expression matching)
    LdFil(OUT). Files that matchs with the pattern recupered from all the list of path
    Returns
    Error code (0 if OK)

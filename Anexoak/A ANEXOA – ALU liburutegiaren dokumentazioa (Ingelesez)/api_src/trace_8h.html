Declare un ordre de trace. Declare au module TR une condition de trace.

  • TRsetTrace donne l'ordre de tracer le module 'modTr' pour le mot-cle 'key' si celui-ci est appele du module 'modApp'. Il est possible de remplacer chacun des termes par NULL ou "all"; dans ce cas, toutes les valeurs de ce terme seront prises en compte.

ALUMCR_EXP int TRtrace ( char *  mod,
char *  key,
char *  function,
char *  pattern,
  ... 
)

Declaration d une trace.

Declare une ou plusieurs traces (module,key). sachant que keys est une chaine de caracteres de une ou plusieurs cles separees par des espaces.

  • Exemple : On se place dans le module 'GC' au niveau de la fonction GCexec. on ajoute

    TRtrace("GC","entry","GCexec"," procCall = %s\n",procCall);

    puis, on decide d activer cette trace a partir d un programme de test. Dans ce programme , on ecrira :

    TRsetTrace("","GC","entry");

    A l'execution, cela donnera : trace GC/GCexec/entry/ >>>> procCall = TRNinsFX

    Si on a : TRtrace("GC","entry parsing","GCexec"," procCall = %s\n",procCall);

    On aura la trace si un des 2 mots cles au moins est actif: Lorsque plusieurs mots cles sont actifs, seule la trace correspondant au premier (de la gauche vers la droite) mot rencontre dans 'TRtrace' est activee.

Parameters
modnom du module dans lequel on se trouve
keynom du mot cle associe a la trace
functionnom de la fonction dans laquelle on se trouve
patternformat de sortie du message de trace
...arguments a tracer, selon le format 'pattern'
Returns
O si OK, -1 si erreur de syntaxe

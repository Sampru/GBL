Automatic action handle and selection for each static visible datum.

Simulate the RETURN action pressing on each visible field of the static data descriptor. By this way, all actions that are executed when the user press the RETURN in the interface are executed. Note that if in the selection values list are more than one line the first of them is selected automatically.

Parameters
CtxDatabase context
LdObjObject data list
LdSdtStatic data descriptor
Returns
Error code (0: OK)

Action handler.

Action handler. Executes an action that has been occurred in a field or in a matrix. NOTE: this function has been created to sustitute the action execution of the criteria, generic and matrix interactors

Parameters
Ctxchannel to database
Actaction to execute
LdObjobject list
LdSdtstatic data descriptor (LdSdt, LdSdtCrt or LdSdtMtx depending on the source field in which has occurred the event)
InxSdtindex of the field in the static data descriptor
InxValindex of the value in which has occurred the event. For static data or criteria this values is 0, for a matrix is the line of the matrix
ClrAllflag that indicates if in the clear action all values of the field must be cleared (static data, criteria) or only current value must be reseted (matrix field)
FldLablabel that is in the field
Mklmake like in the where string or not
Nxt(OUT) next object for navigation
Nrd(OUT) need redraw
LdVal(OUT) value list for lookup
LdLab(OUT) label list for lookup
Returns
Error code (0: OK)

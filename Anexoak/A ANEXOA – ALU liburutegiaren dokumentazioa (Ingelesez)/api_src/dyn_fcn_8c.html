Dynamic lists manipulation functions.

Header file
dynDef.h

Dynamic lists manipulation functions

  • Generalites
    • Ce module fournit des primitives de gestion de liste dynamique.
    • La majorite des fonctions renvoient un entier qui peut prendre 2 valeurs
      • - soit DYN_SUCCES
      • - soit DYN_ERREUR
    • Si une erreur se produit, le message peut etre obtenu par la fonction DYNrendErreur qui retourne la chaine decrivant l'erreur.
    • Toutes ces fonctions sont appelees avec un parametre de type void *, dont la valeur est renseignee lors de l'appel : DYNinit

    • La numerotation des lignes et des colonnes d'une liste debute a 0.

      ou non de la base de donnees.

  • Ces informations sont organisees en colonnes.
  • Chaque colonne contient des informations concernant ses elements : type , taille elementaire, nom de la colonne.
  • Les elements places dans une colonne sont composes des d'informations :
    • - presence d'une information ou pas (flag de validite),
    • - valeur de l'element.
  • Le flag de validite peu prendre les valeurs suivantes :
    • - 0 : donnee valide
    • - -1 : donnee invalide
    • - -2 : donnee de type DATETIME incorrect (mauvais format)
  • Le libelle des colonnes ne depasse pas DYN_MAXCOLNAME (limite 30 car)
  • Certaines fonctions acceptent l'adresse d'une variable destinee a recevoir une information se trouvant dans la liste (ex : DYNgetData).
  • Pour la declaration de cette variable, il est TRES FORTEMENT CONSEILLE d'utiliser les types predefinis (T_DYNretfloat, T_DYNrettiny ...), POUR LIMITER LES PROBLEMES DE PORTABILITE (passage DEC ALPHA).

Adds a column with a value to a dynamic list.

Adds a column with a value to a dynamic list. If the list is NULL this function assumes that it needs initialising with ONE line. Important: The STEP of the new list will be ONE. Think about it. If the list already exists (with 0 or more lines) it resets all the values to the given value/valid (the list may be and continue to be empty).

  • If a column with the same name is present in the list it just reinitialises this value.

  • If we call sequentially twice this function with the same parameters the result will always be the same as calling it once.

    Note that the parameter specification order has been change in respect to DYNajouteColonne and DYNajoutexColonne in order to enable specifying name and value one after the other, for readibility. When more than one column is added, aligning values is encouraged.

Parameters
LdDynamic list, set to NULL if needs initialising

  • Size: (size_t) Size of column, DEV_LENGTH, ...

  • Value: Value, as a constant, while strings are passed as pointers. ex: "ESP", Dev, 12, Qnt, ... Type depends on the column type

Concatene two dinamic lists.

Concatene two dinamic lists.

  • If DYN_NOT_VERIFY_NAME is required, both lists must be identical
  • If DYN_VERIFY_NAME is required, both lists mustn't be identical This choice, this is, DYN_VERIFY_NAME has some special characteristicas that must be taken into account:
  • - columns of Ld2 that does not exist in Ld1, are added to Ld1
  • - columns with different names are treated as different columns (added to the result list).
  • - when two colunms have the same name, their types will have to be compatible, that is: String type columns with different size, will be converted to the greatest size.
  • - columns with same names and different types are considered as an error. If one of the columns is string and the other is a date, it will always be converted to date. (Note that invalid dates may be generated and are not checked). No more conversions are available.
  • In case DYN_VERIFY_NAME is selected, before adding Ld2 data into Ld1 list, the function adds as many empty lines in Ld1 as lines contained in Ld2. This is done to initialise to NULL values all the columns in Ld1 for the lines that are going to be added. The most probably situation is that not all the columns in Ld1 exist in Ld2, so their values must be initialised to NULL values.
  • !!! If you use DYN_NOT_VERIFY_NAME, both lists must be identical !!!

Look up for some information in a dictionary list.

Look up for some information in a dictionary list to add on a data list according to a (possibly composed) key.

  • For each line in the data list we look for additional information on another list, the dictionary, when the key matches ("outer join").
  • If the columns to add don't exist in the data list they will be created.
  • When the columns to look up exist they are NOT initialised to NULL. This means that we may look the information up in different dictionaries.
  • The dictionary list MUST be sorted by the (possibly composed) key, as a binary search is carried out.
  • The function admit joins between NULL values.

Applies where conditions to columns of one dynamic list.

Applies where conditions to columns of one dynamic list and

  • + DELETES the rows that DON'T match the conditions.
  • or
  • + MARKS the rows that DO match the conditions.
  • Intuitively, this is equivalent to keeping those lines that match the where clause in an SQL sentence.
  • Whether the lines are MARKED or DELETED depends on a parameter, ColMrk. If this parameter contains a valid column number it will set a mark on this column. A -1 is considered a bug, i.e. an invalid column number. The DYN_WH_DELETE_LINES value means delete the lines. Note that sometimes is better to mark lines that match with a where clause and then extract those lines with DYNextractMarkedLines that remove then directly wiht DYNwhere.
  • If the function marks the columns, it reinitialises the marked lines each time that it is called, so previous calls to DYNwhere are lost.
  • The general clause and each of the sub-clauses may be negated so that Morgan laws may be applied.

  • NULL values (Validity flag != 0) may only match a non-negated DYN_WH_OP_IS_NULL operator.

    Parameters
    LdDtadynamic list to find the lines that match with conditions
    NegInverse which lines are kept/removed
    • DYN_WH_NOT_NEGATE : Keep rows that match indicated conditions (Keep selected rows in the list)
    • DYN_WH_NEGATE : Keep rows that don't match indicated conditions (Remove selected rows in the list)
    ColMrktwo possibilities:
    • DYN_WH_DELETE_LINES (to delete lines that don't match)
    • Column index to mark
    NbWheNumber of where sub-clauses
    PavFor each sub-clause
    • ColIdx: column index of the first operator
    • Not: if the operator has to be negated or not
      • DYN_WH_NOT : Column conditon NOT complement datas must be obtained
      • DYN_WH_OK : Obtain datas that match indicated column condition1
    • Operator: type of operator to apply.
      • DYN_WH_OP_EQUAL =
      • DYN_WH_OP_GREAT >
      • DYN_WH_OP_GREAT_EQUAL >=
      • DYN_WH_OP_SMALL <
      • DYN_WH_OP_SMALL_EQUAL <=
      • DYN_WH_OP_DIFFERENT !=
      • DYN_WH_OP_LIKE_SENS LIKE case sensitive
      • DYN_WH_OP_LIKE_NO_SENS LIKE case insensitive
      • DYN_WH_OP_IN IN (a Ld, see Value)

      • DYN_WH_OP_IS_NULL = NULL (Valid flag = -1)

        –> For this operator DYN_WH_OP_IS_NULL, the two following parameters are ignored, but a value of the appropriate type must be provided. Otherwise variable parameters may be shifted. See example.

    • ColOrCons: indicates if the value to compare is a column or a datum
      • DYN_WH_DATA_CONST: A constant (recommended)
      • DYN_WH_DATA_PTR: A pointer, as returned by DYNgbGetData
      • DYN_WH_DATA_COL: The index of another column in the list

    • Value: second operator, a data constant ("ESP", 12.3, LdTra, ...) or a column index, depending on ColOrCons.

      For the DYN_WH_OP_IN operator this parameter has to be a dynamic list with the values on the first column of the list.

    Returns
    • DYN_SUCCES : OK
    • DYN_ERREUR : If error

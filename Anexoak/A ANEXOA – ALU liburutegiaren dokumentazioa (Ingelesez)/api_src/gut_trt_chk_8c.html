Treatments check confirmation.

Treatments check confirmation. For each treatment given in the list, it will see if it already exist in treatments auditory table:

  • - If exists, it means that it was already done and only the comment is updated
  • - If NOT exists, it means that it done now and a new line is added in the treatments auditory table
  • The given list must have at least the following columns:
  • - 'Tty': Treatment typ, this is, if it si pre-defined (GUT_TRT_CHK_PDF) or user-defined (GUT_TRT_CHK_UDF)
  • - 'Trt': Treatment code. This column will have only any value in case the treatment type is pre-defined (GUT_TRT_CHK_PDF), otherwise it will be NULL.
  • - 'Des': Treatment description. This column will have only any value in case the treatment type is user-defined (GUT_TRT_CHK_UDF), otherwise it will be NULL.
  • - 'Udn': User who has done the treatment
  • - 'Ddn': Date when the treatment has done
  • - 'Hdn': Hour when the treatment has done
  • - 'Cmt': Comment about the treatment. It can be not valid.

Treatments check scheduling dates confirmation.

Treatments check scheduling dates confirmation. For each treatment given in the treatments list, it will see if there are dates configured in the LL list of dates and in that case, it will delete the previous existing ones and will insert the new ones.

  • The treatment list must have at least the following columns:
  • - 'Grp': Group of treatments
  • - 'Tty': Treatment typ, this is, if it si pre-defined (GUT_TRT_CHK_PDF) or user-defined
  • - 'Trt': Treatment code. This column will have only any value in case the treatment type is pre-defined (GUT_TRT_CHK_PDF), otherwise it will be NULL.
  • - 'Des': Treatment description. This column will have only any value in case the treatment type is user-defined (GUT_TRT_CHK_UDF), otherwise it will be NULL.
  • The dates LL list must have at least the following columns:
  • - 'Grp': Group of treatments
  • - 'Tty': Treatment typ, this is, if it si pre-defined (GUT_TRT_CHK_PDF) or user-defined
  • - 'Trt': Treatment code. This column will have only any value in case the treatment type is pre-defined (GUT_TRT_CHK_PDF), otherwise it will be NULL.
  • - 'Des': Treatment description. This column will have only any value in case the treatment type is user-defined (GUT_TRT_CHK_UDF), otherwise it will be NULL.
  • - 'List': Sublist with the dates of the treatment

int  NbStr,
double  Number,

Given a floating point number obtain the corresponding formatted string.

Given a floating point number obtain the corresponding formatted string Note1: In order to avoid precision problems like 4.9999999999999 -> 4.99 an Epsilon of 10 ^ -(NbDec + 3) is added to the original number. Another Epsilon of 10 ^ -(NbDec + 2) is used to break ties in methods 0 and 2. Note2: To gain precision a double is used as reference instead of a long. This implies the use of the function floor () with the corresponding loose of efficiency. Note3: To have precision with a big number of decimals the number is round after the integer part of the number is got.

Parameters
CpStrReserved character string to hold the result.
NbStrNumber of characters reserved.
NumberNumber to be formatted into a string.
FormatUse of commas and points or points and commas. 0: American, formatted, e.g. 3,159,124.4238 1: European, formatted, e.g. 3.159.124,4238 2: American, no separator for thousands, etc e.g. 3159124.4238 3: European, no separator for thousands, etc e.g. 3159124,4238
NbDecNumber of decimals.
MethodRounding method. 0: Round to the nearest, 9.231 -> 9.23, 9.236 -> 9.24, 9.235 -> 9.24 1: Truncate, 9.231 -> 9.23, 9.236 -> 9.23, 9.235 -> 9.23 2: Up next, 9.231 -> 9.24, 9.236 -> 9.24, 9.235 -> 9.24
Returns
Error code
  • 0: Success. The formatted number copied in CpStr
  • -1: Incorrect method. An empty string is obtained.
  • -2: Incorrect format. An empty string is obtained.
  • -3: Does not fit. The largest possible string is copied. It may be cut anywhere. Used in FMTCOPYTOSTR
  • -4: Not a number or number. It return NaN
  • -5: Number too big. It return +++

ALUMCR_EXP char* FMTformatNum ( char *  valIn,
int  fix,
int  ft,

Cette fonction formate un nombre represente par une chaine de caracteres.

Cette fonction formate un nombre represente par une chaine de caracteres, suivant le format f (voir .h pour plus de precision) et tronque ce nombre a fix decimales apres la virgule.

  • Formate un nombre suivant le format passe en parametre, et l'arrondi a un nombre donne de decimales.
  • Le format est defini de la maniere suivante :
  • F1 + F2 + F3 ... + Fn ou les Fi sont choisis
  • parmis :
  • FCOMMAPOINT : N = x,xxx,xxx.xxx
  • FPOINTCOMMA : N = x.xxx.xxx,xxx
  • FBLANKPOINT : N = x xxx xxx.xxx
  • FBLANKCOMMA : N = x xxx xxx,xxx
  • SIGNED : affichage du signe
  • PARENTHESED : mis entre parentheses
  • PARENTHESED_IF_NEG : mis entre parenthese (sans le signe) si negatif.
  • Toutes ces options sont cumulables. Mais attention, si l'on cumule les options definissant les separateurs (FCOMMAPOINT etc.), un seul sera pris en compte : l' ordre de priorite etant defini par la liste ci-dessus (FCOMMAPOINT > FPOINTCOMMA ,etc.).
  • En outre, il est possible d'exprimer le nombre en entree sous la forme xxxxxxxxxx.xxx ou xxxxxxxxx,xxx et de le faire suivre par une quantite variable de k, K, m ou M tels que: chaque k ou K represente un facteur 1 000 chaque m ou M represente un facteur 1 000 000
  • Tous les espaces apparaissant dans le nombre sont enleves.
  • Les zeros a gauche de la mantisse sont aussi enleves.
  • La partie decimale, lorsqu'elle est fixee, est eventuellement completee par des zeros. ( ex.: fix = 2; 635 devient 635.00)

  • Si l'on passe le format en negatif, la manip inverse est realisee.

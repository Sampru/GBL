Detailed Description

Question management.

This module provides a framework to organise questions.

  • The typical use of this module may be described as follows:
  • 0) Set a question object in the object list
  • - QUEsetLd (&LdObj, QUE_SINGLE, "Question1", "Question", LdAns);
  • - The list LdAns will be destroyed when calling OBJdestroy, so the
  • user must not destroy it.
  • 1) When needed, check if there is any question:
  • - if (QUEisThereAnyQuestion ())
  • - ...
  • 2) If there is a question, show is:
  • - ALERTquestion (LdObj);
  • 3) To get the possible asnswer(s), get the question object, and get
  • the answers of the questions:
  • OBJgetVal (LdObj, "Question", &LdQue);
  • NbQue = DYNgetNbLignes (LdQue);
  • for (InxQue = 0; InxQue < NbQue; InxQue++) {
  • QUEgetLdAnswer (LdQue, InxQue, &LdAns);
  • .....
  • }
  • - The list LdAns will be destroyed when calling OBJdestroy, so the
  • user must not destroy it.
  • - This list LdAns contains two fields:
  • ans: label of answer (char *)
  • mrk: Flag if the answer has been selected. 0: no, 1: yes
  • We will se an example to understand better the way to use this module.
  • The following code shows how we can insert a question in a function. The developper has
  • to verify if a question exists and treat it if so or insert it if it doesn't exist.
  • if (QUEisThereAnyQuestion (LdObj)) {
  • * Verify if this check has already been carried out: 2nd time *
  • if (QUEisThereIdQuestion (LdObj, "DLE")) {
  • ERRchk (OBJgetVal (LdObj, "Question", &LdQue));
  • ERRchk (QUEgetIdLdAnswer (LdQue, "DLE", &LdAns));
  • ERRchk (DYNfirstMark (LdAns, DYNcbn (LdAns, "mrk"), &CntAns));
  • Ans = (char *) DYNgbGetData (LdAns, DYNcbn (LdAns, "ans"), CntAns);
  • if (strcmp (Ans, "No") == 0) {
  • ERRchk (OBJdelObj (LdObj, "Question"));
  • goto ERRend;
  • }
  • ERRchk (QUEdelIdQuestion (LdObj, "DLE"));
  • }
  • }
  • else {
  • * First time *
  • ERRchk (QUEsetYesNo (&LdObj, "DLE", Question, "Yes", "No"));
  • }
  • And the next example tries to show the way to call to a function that has a question
  • into it. This function will be called until all the questions will be answered. The
  • developer has to be careful with the questions manipulation (when insert a question, when
  • delete it, when recover an answer, …). This part of code must be in a CC file because it
  • shows a window with the question.
  • * Treatment executed while the questions are answered *
  • do {
  • ERRchk (BUTexecPreviousFunction (g_gc_ctx, LdObj, Line));
  • if (QUEisThereAnyQuestion (LdObj)) {
  • QUEdone = 0;
  • ALERTquestion (LdObj);
  • }
  • else {
  • QUEdone = 1;
  • }
  • } while (!QUEdone);

Inserts a question in the object list.

Inserts a question in the object list. The inserted question will be another object 'Question' in LdObj. The looK and feel of the question will depend on the parameter Type, which can have the values:

  • - QUE_YES_NO
  • - QUE_SINGLE
  • - QUE_MULTI
  • The possible answers must be in LdAns. This list must have obligatory a column 'ans' with the answers, and optionally another column 'mrk' which will be used to mark selected answers. If 'mrk' column does not exist in LdAns will be created.
  • The list LdAns will be destroyed when calling OBJdestroy, so the user must not destroy it.
Parameters
LdObjPointer to an object list
TypeType of question
QueIdQuestion identifier
QuestionLabel of the question
LdAnsList of answers
Returns
Error code (0: OK)

Detailed Description

Control the dynamic reservations of memory.

This module implements a control of dynamic memory reservations. It allows to keep the list of allocated blocks to verify desallocations, and free a group of blocks as Pascal does, with mark and release.

  • Generalites: The module describes the way to manage the list. It uses two lists and depending on the type which has been reserved, the structure that has all the information about the block, will be stored in the corresponding list. That's made because managing the lists this way, it is faster when the block has to be freed or reserved them again (realloc). 
     When a block is reserved with MMmalloc, the information of the block will be stored in a
    T_MemoryBlock type structure. After doing this, it will take the decision to store this block in one of the lists of T_MemoryList, depending on the type of data that has been reserved. 
     While freeing the block, the programmer has the chance to execute two different funcions,
    and the decision will be taken depending on the type of data that is going to liberate. The only difference between both functions is the order they use while finding the block. If the type of data is a typically reallocated one (void, char, int, short, T_DYNtete, HDBC, RETCODE...) MMfree must be used (it first finds in the reallocated list). Otherwise, the executed method will be MMfreRlc (the inverse order, first not reallocated list). 
     When a block is reallocated, as all the blocks that are reallocated are in the same list,
    it will directly find the block in the typically reallocated block list. This way, the reallocation will be faster.

Re-alloue un bloc.

Re-alloue un bloc en utilisant le gestionnaire de memoire qui doit etre initialise au prealable. En principe, cette fonction n'est pas utilisee directement: il vaut mieux utiliser la macro MMrealloc qui se charge de passer le numero de ligne, le nom du fichier et le type de donnee et eventuellement le thread de maniere automatique cette fonction se comporte comme l'appel systeme realloc

Parameters
pt_oldBlocadresse de l'ancien bloc qui doit etre realloue
newTaillenouvelle taille
ididentifiant (numero) du thread
lignenumero de la ligne source ayant fait l'appel
filenom du fichier ayant fait l'appel
typenom du type de donnee allouee
Returns
  • cette fonction se comporte comme le 'realloc' systeme, c-a-d :
  • - si le bloc etait NULL, fait une allocation
  • - copie les donnees de l'ancien bloc dans le nouveau (si l'ancien etait != NULL)

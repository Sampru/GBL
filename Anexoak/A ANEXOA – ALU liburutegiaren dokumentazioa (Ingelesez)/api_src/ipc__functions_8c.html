Detailed Description

Interprocess communications primitives through sockets.

Acronyme
IPC, InterProcess Communication

This module handles high level socket interprocess communication primitives.

  • So far only AF_INET sockets are considered, and within these SOCK_STREAM (streams: sequenced, reliable, two-way connection-based byte streams) and SOCK_DGRAM (datagrams: connectionless, unreliable messages of a fixed, typically small, maximum length).
  • Specific system functions used/related:
  • accept ()
  • bind ()
  • connect ()
  • close ()
  • getsockname ()
  • listen ()
  • sendto ()
  • socket ()

Read a message from an opened and bound socket.

Read a message from an opened and bound socket.

This function limits the size of the message to 1023 bytes.

It guarantees that in an error free scenario, the byte following the last byte of the message is null ('\0'), makes it suitable for using it with character strings.

It allocates the memory for the message, which must be freed through MMfree.

Parameters
SockIdSocket identifier.
MessRead(out) Pointer to the result (must be MMfree-d)
NbRead(out) Number of bytes read
Note

static int lcPRNCgenRep ( void *  Ctx,
void *  LdDtaRep,
char *  CrtRepFilNam 
)
static

It executes the generation of the report and it prints it if it is necessary.

It executes the generation of the report and it prints it if it is necessary.

  • If you want to obtain the automatically generated report name with fox date and hour, the CrtRepFilNam parameter must be initialized to NULL.
  • Also, it is possible to indicate the name of the created report file in the CrtRepFilNam parameter. If this parameter value exists, it is used as as report file name.
  • NOTE: It returns in CrtRepFilNam parameter, a pointer to a static variable, so the second time that is used it overwrites the first time value.
  • The actions that are carried out in the function are:
  • - Initialise local variables of the module if it is necessary.
  • - Get the name of equivalent report. If equivalent report doesn't exists take the report's name
  • - Determine the type of report (C, PRR or SQR)
  • - Wait for possible zombie process or if a recent report (launched by a fork) is in process
  • - If the current report has to be executed with fork it is made
  • - If the fork has given an error, or the fork has not been executed or it is in the son process:
  • - If it is a SQR report
  • - Look for the form in the sqr file of the report
  • - Get the print commands for the current report
  • - Get the user print commands for the current report
  • - If it is a SQR report calls the function that generates the SQR report
  • - If it is a C report:
  • - Take the current connection to database
  • - Make a new connection to the database specified to pick up the data
  • - Change connection
  • - Call the function to generate C report
  • - Return to the previous connection closing the one that has been open
  • - If it is a PRR report:
  • - If the file has been correctly created and if it is necessary to print, send the impression.
  • If it is the son process free the used memory and kill the process
  • - If it is not necessary to print and it is the son process free the used memory and kill the process

  • - The father continues and frees the used memory.

    Parameters
    CtxDatabase context
    LdDtaRepList with all the necesary data to generate and print the file
    CrtRepFilNam(IN/OUT) Created report file name. Reserve size of memory for this variable (PRR_REP_FIL_SIZ).
    Returns
    Error code (0 if OK)
    Algorithm
    • init the local variables if they are not initialised
    • get the equivalent report name if there is one
    • if there is not an equivalent report name takes as name the report's name
    • look for the report's equivalent name in the list of C report's functions. If the report's name has not been found a SQR report is launched.
    • wait for other zoombie or process forked
    • get if the report has to be forked or not and it forks if it has to be forked
    • if the report is SQR look's for the FormName ("IMP_80", "IMP_132",...) in the sqr skeleton
    • get the print commands and the user print commands
    • depending on the report type:
    • - SQR: generates the report file launching the SQR method
    • - C: generates the report file launching the C method
    • - PRR: generate the report file launching the PRR method
    • if the report file is generated and if it has to be printed it send it depending on if we have to print only with user commans, general print commands or with both types.

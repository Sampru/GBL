Detailed Description

ALU GLoBal Variable ManageMent by ProCeSs module functions.

Header file
glbDef.h
Acronyme
GLBvmmPcs (GLoBal Variable ManageMent by ProCeSs)

This is a module which is used to manage the use of variables to have not problems with the access to them from distinct processes. (For example from main process and from one thread launched from main process)

  • The philosophy of this module is the following:
    • Each process has a copy of each variables which it needs.
    • This variables are stored in a system which manages their values and you retrive the variable which you need in the moment having into account an index which you must initialise the first time that you store the variable.
    • To know which is the index of system in which we have stored the variable, we have a list of variables (lcLdVbl) with three columns: "IdeVbl", "IdeSys". – IdeVble: Identifier of variable (Ctx, Cst list, etc...) – IdeSys: Index of system in which we have stored the variable. – NbPcs: Number of process which are using the system.
    • To manage this system, now we are using a TLS windows functions. you can get more information about TLS in microsoft.com page. ('Using Thread Local Storage' page)

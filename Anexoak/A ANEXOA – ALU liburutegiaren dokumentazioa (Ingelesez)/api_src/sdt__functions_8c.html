Detailed Description

SDT, Static Data (lists, simple tables)

Static data managing functions. These function provide a generic framework to manage simple static data windows and their functions.

  • The code in this file drives the interface on its duty. Decisions must be taken at this level, so that if the interface generator (now ILOG VIEWS) is changed everything need not be reimplemented.
  • Details:
  • A new static data screen NEW without particularities normally requires:
  • Add the call to the menu descriptor(s) MENU. File: mnu_dat_MENU.dat
  • Add its general information. File: sdt_des_alldata.dat
  • Create a descriptor with its information. File: sdt_dat_NEW.dat
  • Create a button descriptor. File: but_dat_NEW.dat
  • Add new key used table information. File: kut_dat_alldata.dat
  • Add the window in the access mode table. File: trtm_mod.dat
  • And optionally:
  • Add new enumerated types. File: enum.dat
  • Add new selection sentences. File: sqls.dat
  • Add calculations. File: calc_grp.dat
  • Add controls. File: ctrl_grp.dat

  • Doubles:
    • Format: Use of commas and points or points and commas.     • NbDec: Number of decimals. The default is in cst.ide = 'DblNbDec'
    • Method: Rounding method. The default is in cst.ide = 'DblRound' 0: Round to the nearest, 9.231 -> 9.23, 9.236 -> 9.24, 9.235 -> 9.24 1: Truncate, 9.231 -> 9.23, 9.236 -> 9.23, 9.235 -> 9.23 2: Up next, 9.231 -> 9.24, 9.236 -> 9.24, 9.235 -> 9.24
    • Remove Zeroes: Indicates that the zeroes are been removed. The default is in cst.ide = 'DblRmZer'
  • Dates:

    • Format: Date style. Is not parametreable. It uses the DTgetFmt () function to get the format. 5 = European 10 = American

Writes a data list into the engine's data descriptor. It changes matrix all columns. All matrix data are removed before inserting the data list. It changes matrix all columns. This function removes the graphic objects of the matrix, so it must be recreated (the matrix columns that ARE NOT in the new values list are RESETED).     • Method: Rounding method. The default is in cst.ide = 'DblRound'
      • 0: Round to the nearest, 9.231 -> 9.23, 9.236 -> 9.24, 9.235 -> 9.24
      • 1: Truncate, 9.231 -> 9.23, 9.236 -> 9.23, 9.235 -> 9.23
      • 2: Up next, 9.231 -> 9.24, 9.236 -> 9.24, 9.235 -> 9.24
    • Remove Zeroes: Indicates that the zeroes are been removed. The default is in cst.ide = 'DblRmZer'
  • Dates:
    • Method: Method used. This Method is used to complete the date
      • 1 = date <= refDate
      • 2 = date >= refDate
      • 3 = date > refDate
    • Format: Date style. Is not parametreable. It uses the DTgetFmt () function to get the format
      • 5 = European
      • 10 = American

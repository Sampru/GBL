Detailed Description

Error management.

Error management

  • Defined macros: ERRdef, ERRchkNoTrace, ERRass, ERRassErr, ERRchk, ERRchkErr, ERRchkNot, ERRret, ERRchkGb, ERRchkDyn.
  • NOTE: ERRchk and ERRass are defined macros which set ErrCd value and go to ERRend if ErrCd value is not zero. They both have no diferences, unless NOASSEPT compilation option is defined. In that case, all ERRass () calls are converted into nothing, so that the execution will be faster. So do careful if you use a function into ERRass because if you define NOASSESPT compilation option, all code will desappear. The same case occurs with ERRassErr and ERRchkArr macros.
Author
Anjel ETXEBERRIA - 03/Mar/1998

Set the DYN error message in the error message list.

If an error code passed as parameter is not null, it checks if a DYN error has occurred and writes the error message into the error list. This function is normally called through the macro ERRchkDyn.

  • In the Ld parameter the pointer to the list is passed instead of the list for when the list is initialised in the DYN called function. Depending on the operating system the Ld variable is updated or not, so we pass the direction because its contents is always updated.

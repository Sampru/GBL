Detailed Description

General functions to manage ALU's web services arguments.

The web services engine allows you to pass the arguments to the web service directly. But there are some cases in which the parameters are complex. For example, when we have parameters of different concepts or some of them are lists and another ones not.

So, there is another possibility to pass the arguments through database for a process identifier. For each call, a process identifier is needed and the arguments must be inserted in the database for this process identifier.

The system gives the following JAVA web services to do it:

  • aluWbsPrcCre: to create a process identifier
  • aluWbsPrcInsArgStr: to insert string argument
  • aluWbsPrcInsArgInt: to insert integer argument
  • aluWbsPrcInsArgFlo: to insert float argument
  • aluWbsPrcInsArgDat: to insert date argument

This module gives the C functions to get the parameters in their original type.

  • ALUwbsArgGetLdPar4WbsPrc: get the list of parameters for an argument and a given process identifier. The resultant list data are in the format than they are inserted. Currently treated data types are:
    • DYN_STRING: string data type which maximum size is WBS_STR_ARG_LEN
    • DYN_INT: integer data type
    • DYN_FLOAT: float data type
    • DYN_DATE: date data type which maximum length is CDA_LENGTH

We can remove data associated to a call using the following function

  • ALUwbsArgDel4WbsPrc: Deletes the information related to a process identifier

To have a correct maintance of the tables related to this module, in the applications purge function, we should add a call to:

  • ALUwbsArgPurWbsPrc: Purges the old information related to web services arguments by process identifier

To use this ALU functionality you need to add alu.core.jar, alu-type.jar, alu-webservices.jar libraries ALU's jar libraries list. And link application's WBS executable with AluWbs.lib library

Calculate a TMP average rate in a date.

Calculate a TMP average rate in a date for estimating the forward interest of a variable repo or loan.

The method builds the Tar (TMP average rate) curve by calculating the TMP forward rates after a yield curve.

The first time this function calculates values for all the dates from ValDat to DfcDat. Further calls may find the target date calculated. If this is not the case all dates from the last calculated to the current one will be calculated.

If the period has begun (Bdt <= Valdat) the rate calculate will be directly taken, else the froward from Bdt to DfcDat will be calculated

If TAR object does not exist, create it. To destroy a TAR object use function MKTtarFree

Calculate the discount factor for a cms point.

Calculate the discount factor for a cms point A CMS (irs or ois) point has coupons and the flows of those coupons must be considered according to the following formula: 

 n-1  Rn * Ni

DFtn - SUM ----— * DFi i=1 B * 100 DFn = ------------------------— Rn * Nn 1 + ----— B * 100

  • Where DFtn = Discount factor of the T/N point if exists, otherwise 1.0
  • R = rate
  • N = number of days between consecutive points
  • B = basis
  • DF = Discount factor of previous points
  • n: index of the point to calculate (Inx)

  • i: index of previously calculated points

    CalDatCalculation date.
    InxIndex of the point to calculate its DF
    ValDfcDF from the reference date till value date (DF of ON or TN, or 1.0)
    Dfc(out) Discount factor or the point.

    Error code (0: OK)

Calculates directly the discount factor of a point

  • There are two methods to calcuate a direct discount factor and these are formulae
  • 1
  • Proportional --------------------—
  • ( Rat * NbDays )
  • ( 1 + -----------—)
  • ( 100 * Basis )
  • 1
  • Actuarial ----------------------------—
  • ( Rat ) (NbDays / Basis)
  • ( 1 + --—)
  • ( 100 )
  • Where
  • + Rat = Annual interest rate
  • + NbDays = Number of days between begin and end date of period
  • + Basis = Number of days we consider a year has

  • This value must be multiplied by the discount factor of the value date. The value date is the end date of the Tom/Next point, because the begind date of a point is the end date of the TN point. If TN point doees not exist will use the OverNight point. And if this does not exist either, then ValDfc must value 1.0

Calculate discount factor of EoY points when there is a value for EoY. We wil apply following formulae 

                (d4 - d3)      Bas       (        (d3 - d2) )
   Zc4 + Zc1 * -----------  -  ---  * Ln (1 + R * --------- )
                (d2 - d1)      d3        (          Bas     )

Zc2 = -------------------------------------------------------------— d2 (d4 - d3) — + ------— d3 (d2 - d1)

Zc2 * d2 Bas ( (d3 - d2)) Zc3 = ------— + --— * Ln ( 1 + R * ------—) d3 d3 ( Bas )

Where

  • Zci: Zero coupon of point i
  • di: Number of days of point i
  • R: Type of the EoY point
  • Bas: Basis of the EoY point
  • Ln: natural logarithm

  • 2: Refers to the first EoY (the last open date of current year)

Controls yield curves.

Controls yield curves. We check that all the required curves have been loaded with correct values. All curves that are not used will be destroyed, so that we don't need to calculate it.

  • In the stock object (LdTra) all curves come in the "crv" and "crvEst" columns.
  • Forex deals: they have an additional curve, which comes in the "crm" column. There is also a Forex reference curve in the "Frf" object of LdMkt which must exist in the presence of forex deals.
  • Repo curves: There are GENERIC and SPECIFIC repo curves. The specific curves may not exist, in which case we use the generic one. The mechanism is as follows: if a curve exists with the name of the bond contract (which comes in the "ctr" column of the LdBndr object) this is the repo specific curve, the one that will be used. Otherwise we need the generic curve (which comes in the "crv" column). If a repo curve exists it must have the correct values (i.e. there may not be NULL or zero value points).
  • Bond prices: In order to calculate the bond prices with their spread, the curve of the contract is needed. It is set in LdPri by the MKTpriGet function. We take it too.
  • Cps and Cpi curves: to calculate those curves they require that their CPS yield curves are already calculate, so those CPS yield curves are also needed.
  • MC instruments: We need the rate interest curve of the instrument and the forward curve of the assets when they are interest rates.
  • For reasons above explained, before calling to this function it must be done the check of bond prices (MKTpriChk), CDS curves (MKTcdsChk) and CDSI curves (MKTcdiChk). Otherwise, we will keep in our structure more yield curves than the extrictly needed with a waste of time and performance and the generation of warnings of missing market data that are not really needed.

Calculate the DF of the begin date of the first MMfuture point.

Calculate the DF of the begin date of the first MMfuture point

Algorithm
:
  • Get the begin date of the future point
  • Search for a valid point whose payment date is after the begin date of future point with a valid rate and which it is not a MMfuture point
  • Search for a valid point whose payment date is after the begin date and it has already a valid discount factor
  • The best point between two points found before it will be the lower one, that means the one that is already calculated and it is nearest from our current point
  • If such a point does not exist
    • Find the first previous point with a valid DF
    • Obtain the zero coupon (Zcp) of previous point
    • With that Zcp, calculate the DF in the begin date of future point
  • else
    • If the valid point is after current point its discount factor has not been calculated yet,
      • Calculate the DF of the next point
    • Interpolate the DF of the begin date of the future point

Controls yield curves and sets warning messages in case the needed market data is missing.

Controls yield curves. We check that all the required curves have been loaded with correct values. All curves that are not used will be destroyed, so that we don't need to calculate it.

  • In the stock object (LdTra) all curves come in the "crv" and "crvEst" columns.
  • Forex deals: they have an additional curve, which comes in the "crm" column. There is also a Forex reference curve in the "Frf" object of LdMkt which must exist in the presence of forex deals.
  • Repo curves: There are GENERIC and SPECIFIC repo curves. The specific curves may not exist, in which case we use the generic one. The mechanism is as follows: if a curve exists with the name of the bond contract (which comes in the "ctr" column of the LdBndr object) this is the repo specific curve, the one that will be used. Otherwise we need the generic curve (which comes in the "crv" column). If a repo curve exists it must have the correct values (i.e. there may not be NULL or zero value points).
  • Bond prices: In order to calculate the bond prices with their spread, the curve of the contract is needed. It is set in LdPri by the MKTpriGet function. We take it too.
  • Cps and Cpi curves: to calculate those curves they require that their CPS yield curves are already calculate, so those CPS yield curves are also needed.
  • MC instruments: We need the rate interest curve of the instrument and the forward curve of the assets when they are interest rates.
  • For reasons above explained, before calling to this function it must be done the check of bond prices (MKTpriChk), CDS curves (MKTcdsChk) and CDSI curves (MKTcdiChk). Otherwise, we will keep in our structure more yield curves than the extrictly needed with a waste of time and performance and the generation of warnings of missing market data that are not really needed.

  • Forex deals: they have an additional curve, which comes in the "crm" column. There is also a Forex reference curve in the "Frf" object of LdMkt which must exist in the presence of forex deals.
  • Repo curves: There are GENERIC and SPECIFIC repo curves. The specific curves may not exist, in which case we use the generic one. The mechanism is as follows: if a curve exists with the name of the bond contract (which comes in the "ctr" column of the LdBndr object) this is the repo specific curve, the one that will be used.   • MC instruments: We need the rate interest curve of the instrument and the forward curve of the assets when they are interest rates.
  • For reasons above explained, before calling to this function it must be done the check of bond prices (MKTpriChk), CDS curves (MKTcdsChk) and CDSI curves (MKTcdiChk).

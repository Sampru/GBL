Detailed Description

Market data general functions.

Header file
mkt_def.h
Acronyme
MKT

Market data general functions

  • This module and its submodules contain code necessary to manipulate market data.
  • - Retrieve market data from the database
  • - Carry out calculations on the market data
  • - Store market data
  • - Retrieve market data from external sources
  • Data structures are normalised in terms of dynamic lists column names.
  • The principle is that each function needs a certain number of input columns on input lists and doesn't care about the existence of other columns that may be added dynamically anywhere along calculations. This means that any function using must only concentrate on the concerning columns, ignoring other columns. This includes calculation, displaying and storing functions. Of course, the order of the columns in the list may change and no assumptions on this order may be taken further to the local scope of each function.
  • The basic idea is that some functions generate these lists, some functions get information from existing lists, possibly adding new columns with calculated data; finaly, this information will be presented to the user or stored in the database.
  • For each function the fields utilised (input) and the fields modified (output) will be specified.
  • It is important to guarantee that independent from the origin of the data the type in the list is homogeneous everywhere.
  • Function organisation:
  • For any market data component to be accessible there are three phases:
  • Get: Get data from the database, normally not stock-sensible. DATABASE ACCESS may ONLY be carried out WITHIN THIS PHASE.
  • Chk: Check if all data necessary for a given stock are present; Remove unnecessary data; Add information present in the stock data structures.
  • Cal: Carry out calculations
  • For each data type, there can be some order to call these functions,
  • e.g.: + MKTpriChk will be called before than MKTyldChk
  • + MKTyldCal will be called before than MKTpriCal
  • + ...

  • Table: mkt_mcd
  • Description: Market data code definitions. This table contains the definition of all internal market data codes (imc), including user mcd-s (umc). It also contains information to identify each market data element, such as the type (typ) and the code (cod), which may the code for a contract, a curve, a matrix, etc.
  • When this information is not enough to uniquely identify each datum we use additional information such as pty/pnt/mat/unt for curves and ech/stk/cpu for option prices and volatilities.
  • imc codes must be unique. To guarantee so we need to guarantee that its construction algorithm leads to a unique code. This function is centralised in mktMcd.c:MKTmcdSetImc ().
  • See also: mkt_dta, mkt_dta_his, mkt_dta_trt_aud, mktMcd.c

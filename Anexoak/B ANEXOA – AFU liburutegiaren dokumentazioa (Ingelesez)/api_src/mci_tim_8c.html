Obtain a basis from the FEA time dependent table standard template.

Obtain a basis from the FEA time dependent table standard template. The done conversions are:

  • "daycount_NU_A0" (Actual/360) -> Bas = "ACT/360";
  • "daycount_NU_A5" (Actual/365) -> Bas = "ACT/365";
  • "daycount_NU_AA" (Actual/Actual ISDA) -> Bas = "ACT/ACT";
  • "daycount_NU_AAISMA (Actual/Actual ISMA Rule 251) -> Bas = "EXA/EXA"; - "daycount_NU_B0" (30/360 U.S. NASD) -> Bas = "030/360"; - "daycount_NU_E0" (30E/360 European ISMA) -> Bas = "03E/360"; - "daycount_NU_F0" (30E+/360 European ISMA) -> Bas = "S30/360";
Parameters
TdtTime table code
Bas(out). Calculation basis. Reserved memory to copy the basis.
Returns
Error code (0 if OK).

Interpolate a date into a time dependent table.

Interpolate a date into a time dependent table. Only the 'step' and 'following' interpolation methods are allowed.

  • 'Step' interpolation prices between two adjacent dates are set equal to the price input corresponding to the earlier date.
  • 'Following' interpolation prices between two adjacent dates are set equal to the price input corresponding to the latter date.
Parameters

FxdFixing date
Val(out). Calculation basis. Reserved memory to copy the basis.

int MCItimGetFeaTim ( void **  LdTim)

Get the list of time dependent tables which are supported by FEA.

Get the list of time dependent tables which are supported by FEA. The format of such strings must be: "daycount_PmtTerm_Basis", where PmtTerm and Basis are strings describing the payment term and daycount basis respectively. The payment term have to be changed by the user, we show it as NU (Number + Unit). The basis string is defined into afuDayCntBsi enumerate. Also the "discount" option has to be added to the list.

LdTim(OUT) Resultant parameters list
Returns
Error code (0: OK)

Get the AFU currency from the Bloomberg currency.

Get the AFU currency from the Bloomberg currency.

  • Bloomberg may return a currency not ISO:
  • AUd is Australian cents: convert into AUD
  • CAd is Canadian cents: convert into CAD
  • CHf is Swiss cents: convert into CHF
  • EUr is Euro cents: convert into EUR
  • GBp is Great Britain pence: convert into GBP
  • SGd is Singapore cents: convert into SGD
  • USd is United States cents: convert into USD

  • ZAr is South African cents: convert into ZAR

    Parameters
    DevCurrency as returned by Bloomberg.
    Ccy(out) FIT currency. Must have reserved memory, of type DEV_TYPE
    Fct(out) Factor to convert values in Bloomberg currency into AFU currency.
    Returns
    Error code (0: OK)

Get the names of new request and answer files for Bloomberg. Including their paths.

Get the names of new request and answer files for Bloomberg. Including their paths.

  • Starting from a generic name for Bloomberg files, get a unique number to add to the filename. The request filename must have an extension of ".req". The answer filename will have an extension ".out", but it is not compulsory by Bloomberg. Finally, add the correspondent paths to the request and answer filenames.

  • The complete name of a file in Bloomberg can not have more than 25 characters. If we append 4 characters for the extension + up to 6 characters for the number, the initial name of the file must be limited to 15.

    Parameters
    CtxDatabase context.
    BlbKeyKey to get an unique number for the request and answer filenames.
    NamGeneric name of the filenames.
    RqtFilNam(out) Filename (without path) of the request filename. Type OSD_STR_TYP.
    AnsFilNam(out) Filename (without path) of the answer filename. Type OSD_STR_TYP.
    RqtFilNamPth(out) Complete filename (with path) of the request filename. Type OSD_STR_TYP
    AnsFilNamPth(out) Complete filename (with path) of the answer filename. Type OSD_STR_TYP.
    Returns
    Error code (0: OK)

Read a Bloomberg bulk field into a list.

Read a Bloomberg bulk field into a list.

  • Find the bulk field in the file.
  • The format of a bulk field is as follows: ";2;10;3;3;04/10/2008;2;57570.00;2;0.00;...;" where ";" is the separator of subfields in a bulk field. The bulk starts and finishes with a separator.
  • First subfield (2) is the number of dimensions
  • Second subfield (10) is the number of lines
  • Third subfield (3) is the number of columns
  • For each line and for each column there are two subfields:
  • + 3: an integer indicating the type of field that follows

  • + 04/10/2008: the subfield with the proper data.

    Parameters
    FilNamFilename (with path) where the bulk field is.
    NbIniLinNumber of data lines before the required line. 0 to get the first data line.
    NbIniFldNumber of fields in the line before the desired bulk field.
    FldFmtField descriptor of the bulk field.
    LdOut(out) Bulk field converted into a dynamic list. Depends on FldFmt.

Put a request into Bloomberg data license server and/or get the answer file.

Put a request into Bloomberg data license server and get the answer file.

  • Use the Java functions given by Bloomberg.
  • Send request file to the Bloomberg host.
  • Wait until the answer file is generated by Bloomberg in the host.
  • Get the answer file and decrypt it.
  • If we are treating a compressed file, the answer file will have an extra sufix ".gz".

  • To do so, we use a java class available from Bloomberg and AFU creates a new class above it and uses an external java binary called BlbAlmSftp.

    ReqFilPthRequest filename, with path.
    AnsFilPthAnswer filename, with path.

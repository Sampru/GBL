Product with cash flows in ech table?

See if the PRODUCT has cash flows in table ech or not. These are: swap, irg, loamul and cd with cash nature and blind options. In same cases the bond repo could also have a schedule in ech, but not always because it depends on if it is configured (there is a flag) in the deal or not. For that aim, this function gives another parameter 'Mfw' (Multiple FloWs), and in case it is "Y", independently on the value of product/nature the returned value will be always 1

Note: the blind contract have 'c' as nature in the contract table so the test for the blind is done before the nature.

PrdProduct
NatNature
MfwA parameter for indicating 'multiple flows' value. It can be empty
Out(out) (1: if the product has cash flows, 0: if the product has not)
Returns
Error code (0: OK)

Convert a bond price in a quotation into the same price in another quotation.

Convert a bond price in a quotation into the same price in another quotation for a given contract and date.

  • The possible quotations (old and new) may be
  • + QUO_STR_YLD: yield
  • + QUO_STR_CPC: clean price (in decimal format)
  • + QUO_STR_DPC: dirty price
  • + QUO_STR_CAM: clean amount
  • + QUO_STR_DAM: dirty amount
  • + QUO_STR_P328: clean price in 328 format
  • + QUO_STR_P320: clean price in 320 format
  • + QUO_STR_P64: clean price in 64 format
  • The convertion may be required in any pair of those quotations.
  • The direct convertions are only:
  • + QUO_STR_CPC -> QUO_STR_DPC
  • + QUO_STR_CAM -> QUO_STR_DAM
  • + QUO_STR_P320 -> QUO_STR_CPC
  • + QUO_STR_P320 -> QUO_STR_CPC
  • + QUO_STR_P64 -> QUO_STR_CPC
  • + QUO_STR_DPC -> QUO_STR_YLD
  • + QUO_STR_DPC -> QUO_STR_DAM,
  • + QUO_STR_CPC -> QUO_STR_CAM
  • + and their inverses.
  • The rest of combinations will make use of intermediary transformations until we reach the desired target.

  • For instance, to convert from QUO_STR_CPC into QUO_STR_YLD: QUO_STR_CPC -> QUO_STR_DPC and QUO_STR_DPC -> QUO_STR_YLD.

Update the list of user defined quotations when a contract is updated.

Update the list of user defined quotations when a contract is updated.

  • If the new quotation selected is the same than the current quotation of the contract and the same than AFU expects (ignoring user defined quotations) there is no need to add a new quotation.
  • Otherwise add user defined quotation for the contract.
  • If the new quotation is different to the current quotation, convert the market prices of the contract into the new quotation.

  • Finally read again in memory the list of user defined quotations to take into account the new quotation defined.

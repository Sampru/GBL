Controls the unicity of the UMC and calculates another one if not.

Controls the unicity of the UMC and calculates an alternative one for the treated contract if repeated. It is only done when the calculation method takes into account the ISIN code. The reason is that there can be multiple contracts (bonds and equities) with the same ISIN code; and their UMC should be unique. It takes the datum saved into the "umc" column as the current umc. If a new one is calculated it is saved into the same column "umc".

Parameters
CtxDatabase context
LdCtrContract data list. There must exist a "ctr" column that contains the current umc.
OldCtrPrevious name of contract.
InxCtrContract data index into the list. -1 for all lines of the list.
Returns
Error code (0: OK)

Get all the existing user market codes for a market data type.

Get all the existing user market codes for a market data type.

  • The retuned list will have the columns:
  • - umc: user market code. In case this value is not valid, it will be initialise with the value of the internal market code ('imc')
  • - imc: internal market code. This column is removed if RemImc = 1
  • - typ: market data typ
  • - cod: FIT internal code
  • - mat: maturity
  • - unt: maturity unit
  • The retuned list will sorted by 'umc'
Global variables
  • App: to see if we are in GV or not. If we are not, it changes connection to hook database

Updates market data codes (internal and user codes) for contracts. It also calculates the UMC taking into account the method defined into the 'umcMth' cst flag.

  • When umcMth is 'isin', get ISIN code from cb_emi.
  • When umcMth is 'code', get contract code from cc_ctr.
  • When calculating the umc; take into account that is possible to have multiple contracts with same ISIN code, so treat it (done only when the calculation method defined with 'umcMth is 'isin').
  • If get market data from Bloomberg add the Bloomberg code for fixed income (ISIN), funds (ISIN) and equity (ISIN + stock market).

  • If the market data from Bloomberg is managed, calculate the Bloomberg code for fixed income (ISIN), funds (ISIN) and equity (ISIN + stock market). This Bloomberg code will not be always calculated:

    • - In case the user has modified it, it will NOT be overwritten although the characteristics, for instance the ISIN has changed
    • - In case the user has NOT modified it, it will be overwritten only if the characteristics change, for instance the ISIN has changed

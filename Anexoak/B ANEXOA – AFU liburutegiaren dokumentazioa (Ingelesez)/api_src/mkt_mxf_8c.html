Detailed Description

Market data - Fra matrixes.

Header file
mkt_def.h
Acronyme
MKTmxf

Market data - Fra matrixes. See mkt.c for details.

  • datastru:
  • Object: Mxd, Fra matrix first start date
  • Name Type Origin Description
  • mtx CUR_TYPE mtx_def.mx_cod Matrix code
  • dev DEV_TYPE mtx_def.mx_dev Currency
  • mth FFX_TYPE mtx_def.mx_mth Method (e.g. FIXED33)
  • lag ITR_TYPE mtx_def.mx_lag Start date lag (only for SPOT mehtod)
  • dat CDA_TYPE datGetListDates () First start date for mth
  • Object: Mxf, Fra matrixes
  • Name Type Origin Description
  • mtx CUR_TYPE mtx_def.mx_cod Matrix code ("LIBOR ESP")
  • dev DEV_TYPE mtx_def.mx_dev Currency ("EUR")
  • mat ITR_TYPE mtx_def.mx_dur Duration maturity (15)
  • unt UNT_TYPE mtx_def.mx_duu Duration unit ("M")
  • mth FFX_TYPE mtx_def.mx_mth Method to get starting dates ("FIXED33")
  • lag ITR_TYPE mtx_def.mx_lag Start date lag (only SPOT method)
  • nbs ITR_TYPE mtx_def.mx_nbs Nb of start points (8)
  • nbd ITR_TYPE mtx_def.mx_nbd Nb of durations (4)
  • mpx COD_TYPE mtx_def.mx_mpx Matrix type ("mfra")
  • bas ACR_TYPE mtx_def.mx_bas Base ("ACT/360")
  • fxp ITR_TYPE mtx_def.mx_fxp 1st line in price
  • par CUR_TYPE mtx_def.mx_par Parent ("MIBOR ESP")
  • LdXax DYN_VOID MKTmxfGetMxd () List of X axis points
  • LdYax DYN_VOID MKTmxfGetMxd () List of Y axis points
  • LdPts DYN_VOID MKTmxfGetMxd () List of values list
  • LdMtx DYN_VOID MKTmxfGetMxd () List of values matrix
  • List: LdXax, X axis points
  • Name Type Origin Description
  • val INT_TYPE MKTcalIntList () Duration, e.g. 90
  • nbj INT_TYPE MKTmxfCal () Number of days 90
  • List: LdYax, Y axis points
  • Name Type Origin Description
  • dat CDA_TYPE lcMKTmxfCalDatList () Start dates
  • m_y char (9) MKTmxfCal () Start dates - MMM-YYYY
  • List: LdPts, values list
  • Name Type Origin Description
  • cod CUR_TYPE MKTmxfCalPtsList () Matrix code
  • imc RIC_TYPE MKTmxfCalPtsList () Internal point imc code
  • val DBL_TYPE MKTmxfCalPtsList () Point value
  • List: LdMtx, values matrix
  • Name Type Origin Description
  • X00 DBL_TYPE MKTmkMtx () 1st data column
  • X01 DBL_TYPE MKTmkMtx () 2nd data column

  • X02 ...

Calculate a forward rate interpolated from the matrix or the hybrids.

Calculate a forward rate interpolated from the matrix or the hybrids In the market data list passed as parameter (LdMkt), it must exist at least an object 'Mxf' containing the matrixes for the calculations.

Parameters
MxfFra matrix code
CrvYield curve code
LdMktMarket data object list
DatCalculation date
BdtBegin date
EdtEnd date
BasCalculation basis
Fwd(out) Forward yield
Returns
Error code (0: OK) algorith:
  • Tij here represents an existing point in the matrix
  • Ti here represents a point already interpolated on between Tij and Tij+1
  • T is the final interpolated forward type
  • Interpolation between T11, T12, T21 and T22 is carried out first between T11 and T12 to get T1 and between T21 and T22 to get T2, and after between T1 and T2 to get T, as follows:
  • 4 cases are treated. Three of them require interpoling.

  • A number represents one of the interpolation cases

    D1 D D2 D1 D D2

    s1 | T11 | | T21 | s1 | T11 | 3 | T21 |
    s | T1 | T | T2 | s | 2 | 4 | |
    s2 | T12 | | T22 | s2 | T12 | | T22 |
  • Extrapolation on the durations axis has been introduced. It is noted that this extrapolation is only valid for durations close to the last duration. Isma - 07/Jun/1999
